## Project Summary

**VoiceNotes AI** is a privacy-first, voice-driven note-taking and task management mobile app built with Flutter. Users record voice notes, the app transcribes audio, auto-detects language, and uses AI to structure content into actions, todos, reminders, and general notes.

**Current Phase:** MVP (Phase 1) — ~15% complete (UI scaffold done, no backend logic)

---

## Core Principles (MUST follow)

1. **Privacy-first** — All data stored locally in encrypted Hive. No cloud storage in MVP. AI interactions are stateless (zero data retention on server).
2. **No login required** — MVP works without any account creation or sign-in.
3. **Voice as primary interface** — Every core feature is voice-accessible.
4. **No ads, ever** — Revenue through freemium model only.
5. **Stateless AI** — All API calls to transcription/AI services are transactional. No memory, no context carried between requests.

## Coding Conventions

- **Dart style:** Follow official Dart style guide and `analysis_options.yaml` linting
- **File naming:** snake_case for all Dart files
- **Widget structure:** One widget per file for pages, can combine small widgets
- **State management:** Use Riverpod providers (not setState for shared state)
- **Database:** All persistence through Hive repositories (never direct Hive access from UI)
- **API calls:** All external API calls are stateless and transactional — no data retention
- **Error handling:** Graceful fallbacks for API failures, show user-friendly messages
- **Privacy:** Never send user-identifying data to external services. Never log sensitive content.

3. **Follow IMPLEMENTATION_PLAN.md** step order — each step depends on the previous.

9. **Privacy is non-negotiable** — local-first, no telemetry, no tracking.
